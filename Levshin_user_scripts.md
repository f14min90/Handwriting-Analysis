## **[ Сценарий 1 - Регистрация / Вход пользователей** ]

   ### **Если пользователь имеет аккаун:**
    
  1. Пользователь переходит в раздел **Вход**
  2. Пользователь вводит в первое из двух имеющихся полей свою электронную почту. Во второе поле - свой пароль. После чего нажимает кнопку *войти*
  3. Приложение отпраляет запрос на сервер с логином и паролем и ожидает ответа - имеется ли аккаунт с такими данными в базе
  4. В случае совпадения приложение предоставляет пользователю доступ к функционалу приложения (описанному ниже). В случае несовпадения сервер возвращает ошибку и предлагает пользователю попробовать повторить попытку входа или восстановить утерянный пороль
  5. Если пароль был утерян и пользователь выбрал опцию **восстановить пароль**, приложение перенаправляет пользователя на сайт HandWriting Analysis, где он может восстановить пароль
  Примеч. : Если пользователь уже осуществлял вход на данном устройстве, то поля *логин* и *пароль* уже будут заполнены, для доступа необходимо лишь повторное нажатие кнопки **Вход**
  
   ### **Если пользователь не имеет аккаунта:**
  
  1. Пользователь переходит в раздел **Вход**
  2. Пользователь нажимает на кнопку **Регистрация**
  3. Приложение перенаправляет пользователя на сайт HandWriting Analysis, где он регистрирует новый аккаунт
  4. После регистрации пользователь нажимает кнопку **Войти** и входит в свой аккаунт по описанной выше схеме

## **[ Сценарий 2 - Загрузка изображений для сравнения ]**

  1. Пользователь переходит в раздел **сравнение изображений**
  2. а) Пользователь нажимает на кнопку **загрузить из галереи** и выбирает изображение из памяти устройства
  3. б) Пользователь нажимает на кнопку **сделать фото** и делает снимок с помощью камеры устройства
  4. Если пользователь ошибся при выборе фотографии, он нажимает непосредственно на неё и повторяет процесс выбора
  5. Пользователь нажимает кнопку **сравнить** и фотографии отправляются на сервер
  6. Приложение ожидает ответа от сервера в виде вещественного числа диапозона **[0; 1]** - вероятность того, что два изображения принадлежат одному и тому же человеку
  
   Примеч.: фотографии выбираются по отдельности, есть возможность выбрать одну из галерии и сделать снимок второй, или наоборот
  
## **[ Сценарий 3 - Добавление изображений текстов в хранилище ]**

  1. Пользователь переходит в раздел **Добавить изображения**
  2. а) Пользователь выбирает изображение, хранящееся на устройстве, нажимая на кнопку **загрузить из галереи**
  3. б) Пользователь делает снимок с помощью камеры, нажимаю на кнопку **сделать снимок**
  4. Если необходимо, пользователь вводит в графу *информация* пометки к загружаемому изображению (например ФИО / дату)
  5. Пользователь нажимает кнопку *Загрузить изображение*
  6. Приложение отправляет запрос серверу, содержащий изображение и введённое пользователем ФИО. Сервер сохраняет данные в базе
  7. В случае успеха пользователь увидит на экране оповещение об успешной загрузке изображения
  
## **[Сценарий 4 - Просмотр изображений текстов в хранилише ]**
  
  1. Пользователь переходит в раздел **Хранилище**
  2. Пользователь нажимает на кнопку **Обновить хранилище**
  3. Приложение запрашивает у сервера все изображения, которые загружал пользователь ранее
  4. В случае успеха приложение отображает первое изображение из списка, его номер, информацию, а также общее число полученных изображений
  5. С помощью кнопок навигации "**<**" и "**>**" пользователь может листать изображения, выбирая нужное
  6. Чтобы скачать изображение, пользователь может кликнуть по картинке и выбрать папку для сохранения в памяти своего устройства
  7. Чтобы удалить изображение их хранилища, пользователь нажимает кнопку **удалить**
  8. Чтобы изменить информацию об изображении, пользователь нажимает на окно, отображающее текст-информацию и вводит новые данные, затем нажимает на кнопку **сохранить**
